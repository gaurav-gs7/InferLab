# Support

InferLab is pre-alpha. It is suitable for development and evaluation, not yet for production traffic.

- Use GitHub Issues for reproducible bugs and scoped feature proposals.
- Use GitHub Discussions for design questions, evidence-adapter proposals, policy semantics, and usage help when Discussions are enabled.
- Use the private process in [SECURITY.md](SECURITY.md) for vulnerabilities or sensitive-data exposure.

When filing a bug, include the InferLab version/commit, Go version, operating system, sanitized configuration, minimal synthetic trace, expected behavior, actual behavior, and exact reproduction command. Never publish real prompts, responses, secrets, tenant identifiers, or proprietary workload data.

There is currently no paid support or response-time guarantee.
