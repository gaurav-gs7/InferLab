# Governance

InferLab currently uses a maintainer-led governance model appropriate for an early-stage project.

## Roles

- **Contributors** participate through issues, discussions, documentation, code, tests, benchmarks, and reviews.
- **Reviewers** are trusted contributors who regularly provide technically sound review in defined areas.
- **Maintainers** set release scope, merge changes, manage security reports, and steward project direction and community health.

The initial maintainer is Gaurav G S (`@gaurav-gs7`). Reviewer or maintainer status is earned through sustained, high-quality contributions, constructive review, sound judgment around compatibility/security, and adherence to the Code of Conduct. Maintainers document additions or removals in this file through pull request review.

## Decisions

Routine changes use pull-request consensus. Significant public API, trace-schema, architecture, dependency, governance, or compatibility changes begin with a public design issue and a review period appropriate to impact. Maintainers make the final decision when consensus cannot be reached and record the rationale publicly.

Security incidents and time-sensitive release integrity decisions may be handled privately until coordinated disclosure is safe.

## Releases

Maintainers approve and sign releases after required checks, compatibility tests, documentation, security review, and artifact verification pass. Release notes distinguish stable, experimental, deprecated, and removed behavior.

## Independence

InferLab integrates with upstream inference projects but is not endorsed by or affiliated with them unless explicitly stated. Upstream contributions must stand on their own technical merit.
